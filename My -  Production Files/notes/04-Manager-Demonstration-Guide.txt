---
HOW TO RUN AND DEMONSTRATE THE PROJECT
---
Because this prototype simulates a secure backend connection, browsers (like Chrome) will block the API client from loading local database files if you simply double-click the HTML file (due to CORS security policies).

To run the project flawlessly:
1. Double-click the `Run-Project.bat` file located in the root of the `My - Production Files` directory.
2. This script automatically starts a local Python HTTP server and opens your default web browser to `http://localhost:8000/College-List-Demo.html`.
3. The UI will instantly load. As you click on different universities in the left-hand sidebar (e.g., Harvard, UT Austin), the UI triggers an API request.
4. The system securely routes the request through the API Gateway, fetches the record from the database, and dynamically re-renders all charts, tables, and statistics.

---
HIGHLIGHTING THE ARCHITECTURE TO STAKEHOLDERS
---
When presenting this to stakeholders, emphasize the following robust features:

1. THE STRICT SCHEMA:
Point them to `database/database_schema.json`. Explain that we discarded the basic AI-generated template in favor of an industrial-grade schema that perfectly matches the 18 holistic factors, academic metrics, and financial structures outlined in the master Excel specification. 

2. THE CONFIGURATION AND API KEYS:
Point them to `database/config.json`. This is the core configuration file that manages database security. Show them the active API Key (`DREAM_COLLEGE_API_v1_847294`). 

3. THE SECURE MIDDLEWARE:
Explain that the HTML file DOES NOT touch the JSON files directly. Instead, it instantiates a client (`api_client.js`) and passes it the API Key. 
- Demonstration Trick: Open `College-List-Demo.html` in a text editor, scroll to the bottom (line ~2830), and change the API Key string. Save the file and refresh the browser. The UI will fail to load the data, and the browser console will show an "Unauthorized" error. Change the key back, refresh, and it works perfectly again. This proves that the API Key Gateway is fully functional!

This frontend prototype is now fully decoupled from static file assumptions. It operates entirely on an API-driven architecture to make your backend integration seamless.

---
1. THE DATABASE SCHEMA EXPECTATIONS
---
The UI strictly relies on a JSON schema derived from the product requirements (`database/database_schema.json`). When you model the PostgreSQL/MongoDB database tables or document collections, you must map your endpoints to output JSON that perfectly aligns with this schema.
- The root must contain exactly 5 objects: `cds_meta`, `cds_admissions`, `cds_academics`, `cds_financials`, `cds_outcomes`.
- The UI DOM-parsing functions (e.g., `renderSchoolData` in `College-List-Demo.html`) expect these exact keys. For example, the GPA distribution logic iterates over `data.cds_admissions.gpa_distribution`. If your backend payload changes this structure, the frontend graphs will crash.

---
2. THE MOCK API GATEWAY
---
Currently, the frontend requests data through a mock gateway located at `database/api_client.js`.
- It exports a class `DreamCollegeAPI`.
- It is initialized with an API Key.
- It exposes an async method `getSchool(filename)`.
- Under the hood, it fetches a local `config.json` to authorize the key, and if valid, it performs an HTTP `fetch` to the local `/database/records/` directory.

---
3. FRONTEND INTEGRATION POINTS

---
4. YOUR BACKEND INTEGRATION CHECKLIST
---
When you are ready to connect the real Node.js/Python server:
A. Delete our `database/config.json`, `database/api_client.js`, and `database/records/` directory. These are strictly for the local prototype.
B. Serve the JSON payload dynamically from your REST API endpoint (e.g., `GET /api/v1/schools/:id`).
C. Inside `College-List-Demo.html`, delete the `<script src="database/api_client.js"></script>` tag.
D. Update the `dbAPI` object in `College-List-Demo.html` to be a real network wrapper. For example:
   ```javascript
   const dbAPI = {
       getSchool: async (id) => {
           const res = await fetch(`https://api.dreamcollege.ai/v1/schools/${id}`, {
               headers: { 'Authorization': 'Bearer YOUR_ACTUAL_JWT_OR_KEY' }
           });
           return res.json();
       }
   };
   ```
E. Ensure your backend endpoint validates the Authorization Bearer Token or API Key.

Because the UI logic is completely isolated from the fetching mechanism via the `dbAPI.getSchool()` abstraction, your integration will require virtually zero changes to the complex UI rendering logic.

Good luck!

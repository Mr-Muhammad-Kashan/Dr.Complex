# Database Schema and API Key Management

## 1. The Database (`database/records/`)
Our database is comprised of 16 individual JSON files. Each JSON file corresponds to a university's Common Data Set (CDS). 
All data inside these files has been strictly formatted to abide by `database_schema.json`.

The schema divides data into:
- `cds_meta`: UI data, rankings, locations, logos.
- `cds_admissions`: Applicants, admitted, enrolled, test scores, acceptance rates.
- `cds_academics`: Student-faculty ratio, enrollment size.
- `cds_financials`: Tuition, room and board, financial aid.
- `cds_outcomes`: Graduation and retention rates.

*Note: The JSON files provided by the manager have been fully integrated, restructured to be 100% compatible with the frontend schema, and verified against internet data for 2023-2024.*

## 3. Data Flow
HTML (`College-List-Demo.html`) -> Calls `dbAPI.fetchSchoolData(id)` -> API Client authenticates -> API Client reads `[School].json` -> Returns payload to HTML -> HTML renders charts and metrics.
